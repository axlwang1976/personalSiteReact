箭頭函式

arrow function

Back

1. ES6 撰寫函式的方法新增了箭頭函式 arrow function，讓程式碼更精簡，撰寫更快速

2. 以下舉例說明

const firstNames = ['飛', '菲'];

// ES5 寫法
const fullNames = firstNames.map(function(name) {
    return '張' + name;
});

// 箭頭函式寫法
const fullNames2 = firstNames.map((name) => {// 箭頭取代 function 字樣，引數寫在箭頭左方，箭頭右方為函式本體
    return '張' + name;
});
const fullNames3 = firstNames.map(name => {// 引數只有一個可省略 ( )
    return '張' + name;
});
const fullNames4 = firstNames.map(name => return '張' + name);// 若函式內容僅有一列，可省略 { }
const fullNames5 = firstNames.map(name => '張' + name);// 若函式內容僅有一列且為 return something，可省略 return 字樣
// fullNames1~5的輸出結果皆相同為['張飛', '張菲']

const fullNames6 = firstNames.map(() => '張灰');// 若沒有引數要寫空白 ( )

3. 沒有特定的 this 關鍵字，函式內的 this 均繼承自父層

4. 何時不要用箭頭函式：要使用 this 時，要新增物件的 prototype method 時，要使用 arguments object 時
